# 临床辅助决策系统（CDSS 3.0）详细需求文档（PRD）

**版本：** v1.1
**日期：** 2026-03-12
**状态：** 已确认

---

## 一、项目背景

医院临床决策对医生的知识调用效率、信息整合能力要求极高。当前医生普遍面临以下痛点：

- 知识分散在不同系统中，查阅效率低
- HIS系统内缺乏即时知识辅助，依赖个人经验
- 检验结果解读、风险评估等需要额外查阅参考书
- 指南更新频繁，医生难以及时跟进

CDSS 3.0 旨在建设一套**围绕医生诊疗行为**的临床辅助决策系统，将知识查询、临床辅助、系统集成、持续更新整合为统一产品。

---

## 二、项目目标

| 目标层次 | 具体描述 |
|--------|---------|
| 核心目标 | 为医生提供诊疗全流程中的知识辅助与决策支持 |
| 产品目标 | 建设多端统一的知识平台，支持 PC 端独立使用和 HIS 内嵌集成 |
| 技术目标 | 构建可扩展的服务层，预留 HIS/LIS/PACS/EMR 对接接口 |
| 运营目标 | 支持知识库持续更新，管理端支持内容运维闭环 |

---

## 三、用户角色

| 角色 | 描述 | 主要使用端 |
|-----|------|----------|
| 临床医生（门诊/住院） | 核心用户，日常查询知识、使用辅助决策 | PC端、HIS嵌入端、App |
| 住院医师/规培医生 | 三基学习、公式使用、量表使用 | PC端、App |
| 系统管理员 | 知识库维护、配置管理、权限管理 | 管理端 |
| HIS集成对接人员 | 外部系统对接配置 | 平台服务层 |

## 四、使用场景

### 场景 1：门诊接诊辅助
医生在 HIS 开具处方时，CDSS 嵌入侧边栏，根据患者主诉/当前诊断，推荐相关疾病知识、治疗方案、检验检查项目，医生可一键查看指南依据。

### 场景 2：独立知识查询
医生在 PC 端查询某疾病的诊疗规范、检验指标参考范围、药品相互作用，进行日常学习与备课。

### 场景 3：检验结果解读
医生查看 LIS 报告时，CDSS 自动解读异常指标含义，给出可能的临床意义提示，减少漏诊。

### 场景 4：风险评估
医生使用量表（如 CURB-65、Wells 评分等）对患者进行风险分层，系统自动计算评分并给出推荐处置意见。

### 场景 5：移动端随时查询
医生在查房或移动场景下，通过 App 快速查询药品说明书、检验参考值、临床指南。

---

## 五、系统端划分

```
CDSS 3.0
├── 医生PC端（独立知识查询端）     → Web 应用，面向浏览器
├── HIS嵌入端（诊疗辅助端）        → 嵌入 HIS 的 iframe/WebView
├── 移动端（App）                  → iOS/Android，后置开发
├── 管理端                        → Web 后台，面向运营人员
└── 平台服务层                    → REST API，为各端提供数据与能力
```

**关键约束：**
- 医生PC端与HIS嵌入端**共用同一套服务层和知识库**
- HIS嵌入端为轻量模式，聚焦诊疗辅助功能
- 管理端独立部署，权限隔离

## 七、MVP 范围

### MVP 核心原则
> 优先建设 **HIS嵌入端全量功能 + 医生PC端基础知识检索 + 管理端最小后台 + 平台服务层基础能力**。
> 外部系统（HIS/LIS/PACS/EMR）全部以 Mock 数据先行，预留真实对接接口。

### MVP 包含

**HIS嵌入端（全量）**
- 6.2 节中所有 23 个功能模块，外部数据以 Mock 实现

**医生PC端（基础）**
- 首页工作台（含分类导航入口）
- 全局检索
- 疾病知识库 + 疾病详情
- 药品说明书查询 + 药品详情
- 检验检查知识查询
- 内容详情展示

**管理端（最小）**
- 疾病库/药品库/检验库基础 CRUD 与批量导入
- 分类导航管理
- 账号权限管理（管理员账号）
- 知识库更新管理

**平台服务层（全量 P0）**
- 6.5 节中所有 P0 服务，外部对接适配器以 Mock 实现

### MVP 不包含（后续迭代）

- 移动端 App
- 真实 HIS/LIS/PACS/EMR 对接（有接口预留）
- 管理端 P1/P2 功能（检索配置、在线更新等）
- 医生PC端 P1/P2/P3 功能（三基、文献库、自测等）

## 八、关键业务流程

### 流程 1：知识检索流程（PC端）
```
医生打开PC端 → 输入关键词 → 调用检索服务
→ 返回跨库结果列表 → 点击进入详情 → 查看相关推荐
```

### 流程 2：HIS嵌入辅助流程
```
医生打开HIS诊疗页 → CDSS侧边栏加载
→ 读取当前患者信息（Mock/接口）→ 展示疾病知识卡片
→ 医生查看药品/指南/检验项目 → 辅助开单决策
```

### 流程 3：检验结果解读流程（HIS嵌入端）
```
LIS返回检验结果（Mock）→ CDSS解读异常指标
→ 展示临床意义提示 → 推荐相关处置建议
→ 医生确认并记录
```

### 流程 4：量表评估流程（HIS嵌入端）
```
医生触发量表 → 系统自动填入患者基础信息
→ 医生补录缺失项 → 提交计算
→ 返回评分 + 解读 + 推荐处置意见
```

### 流程 5：诊断合理性审核流程（HIS嵌入端）
```
医生提交开单/开药动作 → CDSS触发审核
→ 校验诊断与处置一致性 → 发现问题则弹出预警提示
→ 医生确认或修改后继续
```

### 流程 6：知识库更新流程（管理端）
```
管理员登录管理端 → 进入知识库管理
→ 上传数据文件（Excel/JSON）→ 系统导入校验
→ 发布上线 → 检索服务实时可查
```

## 九、非功能性需求

| 类别 | 要求 |
|-----|------|
| 性能 | 检索响应 < 500ms，详情加载 < 1s |
| 可用性 | 服务可用率 ≥ 99.9% |
| 安全 | JWT 认证，接口权限控制，操作审计日志 |
| 兼容性 | HIS嵌入端兼容 IE11+/Chrome，PC端支持主流浏览器 |
| 扩展性 | 服务层按 API 契约扩展，新增知识库不影响现有接口 |
| 数据 | 敏感患者数据不在CDSS系统落地，仅实时传输使用 |

## 十、后续扩展建议

1. **AI 增强**：接入大模型，实现自然语言查询、智能摘要、病历生成辅助
2. **个性化推荐**：基于医生科室/历史行为的个性化知识推荐
3. **移动端**：iOS/Android 原生或 React Native 实现
4. **真实 HIS 对接**：与医院 HIS 厂商（如卫宁、东软等）完成标准接口对接
5. **质量指标**：诊疗行为统计分析，医生使用行为分析报表
6. **知识图谱**：构建疾病-药品-检验-指南关联图谱，增强推荐质量
