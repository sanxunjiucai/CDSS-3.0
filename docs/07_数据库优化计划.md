# CDSS 3.0 数据库优化计划

**版本：** v1.0
**日期：** 2026-03-17
**状态：** 待执行

---

## 一、当前数据库现状评估

### 1.1 已有表（11张）

#### 知识库表（7张）
- ✅ `diseases` - 疾病库（含 treatment_plans 子表）
- ✅ `drugs` - 药品库
- ✅ `exams` - 检验检查库
- ✅ `guidelines` - 指南库
- ✅ `formulas` - 公式库
- ✅ `assessments` - 量表库
- ✅ `literature` - 文献库

#### 系统表（4张）
- ✅ `users` - 用户表
- ✅ `audit_rules` - 审核规则表
- ✅ `audit_logs` - 审核日志表
- ✅ `system_config` - 系统配置表

### 1.2 当前设计的问题

#### 问题1：疾病表设计过于简单
- ❌ 缺少别名表，搜索命中率低
- ❌ 缺少分类表，无法树状管理
- ❌ 症状/检查/药物关联存储在JSON中，无法做规则触发
- ❌ 缺少拼音字段，搜索体验差

#### 问题2：缺少HIS嵌入端核心业务表
- ❌ 无患者表，无法存储患者上下文
- ❌ 无诊断记录表，无法追踪诊疗过程
- ❌ 无检验结果表，无法做结果解读
- ❌ 无处方表，无法做用药审核
- ❌ 无风险评估记录表，无法追踪量表使用

#### 问题3：缺少规则引擎支撑表
- ❌ audit_rules 只能做简单审核，无法支持复杂规则
- ❌ 缺少规则条件表、规则动作表
- ❌ 缺少规则适用范围表

#### 问题4：缺少知识管理支撑表
- ❌ 缺少知识来源表，无法追溯证据
- ❌ 缺少导入批次表，无法追踪导入历史
- ❌ 缺少导入错误日志表，排查困难
- ❌ 缺少版本管理表

---

## 二、优化目标

### 2.1 短期目标（MVP阶段）
支持PC端和管理端完整功能，为HIS嵌入端预留扩展能力

### 2.2 中期目标（第二阶段）
支持HIS嵌入端23个功能，实现真正的临床辅助决策

### 2.3 长期目标（第三阶段）
支持复杂规则引擎、知识图谱、AI辅助诊断

---

## 三、优化策略

采用**三阶段渐进式优化**，避免一次性改动过大：

### 阶段一：知识库增强（P0，立即执行）
优化现有知识库表结构，提升查询和管理能力

### 阶段二：业务表补充（P1，HIS嵌入端开发前）
补充患者、诊断、检验等核心业务表

### 阶段三：规则引擎完善（P2，规则功能开发时）
完善规则引擎表结构，支持复杂决策逻辑

## 四、阶段一：知识库增强（P0）

### 4.1 疾病库优化

#### 4.1.1 优化 diseases 表
**新增字段：**
```sql
-- 搜索支持
pinyin VARCHAR(255)              -- 拼音全拼
pinyin_abbr VARCHAR(64)          -- 拼音简拼

-- 分类增强
disease_type VARCHAR(50)         -- 感染性/慢性/急症/肿瘤/遗传/免疫
is_common BOOLEAN DEFAULT false  -- 是否常见病
is_chronic BOOLEAN DEFAULT false -- 是否慢病
is_infectious BOOLEAN DEFAULT false -- 是否传染病
severity_level VARCHAR(32)       -- 严重程度：轻度/中度/重度/危重

-- 版本管理
version_no VARCHAR(64)           -- 版本号
source VARCHAR(200)              -- 数据来源（已有，保留）
```

#### 4.1.2 新增 disease_aliases 表（疾病别名）
```sql
CREATE TABLE disease_aliases (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    disease_id UUID NOT NULL REFERENCES diseases(id) ON DELETE CASCADE,
    alias_name VARCHAR(255) NOT NULL,
    alias_type VARCHAR(64),  -- 俗称/简称/历史名称/英文名
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
CREATE INDEX idx_disease_aliases_disease_id ON disease_aliases(disease_id);
CREATE INDEX idx_disease_aliases_name ON disease_aliases(alias_name);
```

#### 4.1.3 新增 disease_categories 表（疾病分类）
```sql
CREATE TABLE disease_categories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    category_code VARCHAR(64) UNIQUE NOT NULL,
    category_name VARCHAR(255) NOT NULL,
    parent_id UUID REFERENCES disease_categories(id),
    level_no INT DEFAULT 1,
    sort_order INT DEFAULT 0,
    status VARCHAR(32) DEFAULT 'active',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
CREATE INDEX idx_disease_categories_parent ON disease_categories(parent_id);
```

```sql
CREATE TABLE knowledge_sources (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    source_code VARCHAR(64) UNIQUE NOT NULL,
    source_name VARCHAR(255) NOT NULL,
    source_type VARCHAR(64),     -- 指南/共识/规范/文献/院内

    publish_date DATE,
    effective_date DATE,
```

#### 4.4.2 import_batches 表（导入批次）
```sql
CREATE TABLE import_batches (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    import_type VARCHAR(64) NOT NULL, -- disease/drug/exam/guideline等
```

## 五、阶段二：业务表补充（P1）

### 5.1 患者相关表

#### 5.1.1 patients 表（患者基本信息）
```sql
CREATE TABLE patients (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    patient_no VARCHAR(100) UNIQUE NOT NULL, -- HIS患者编号
    name VARCHAR(100) NOT NULL,
    gender VARCHAR(10),
    birth_date DATE,
    age INT,
    id_card VARCHAR(50),
    phone VARCHAR(50),

    -- 临床信息
    blood_type VARCHAR(10),
    allergies JSON,              -- 过敏史 [{drug:'青霉素',level:'严重'}]
    past_history JSON,           -- 既往史
    family_history JSON,         -- 家族史

    -- 系统字段
    source VARCHAR(50) DEFAULT 'his', -- his/manual/import
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
CREATE INDEX idx_patients_no ON patients(patient_no);
CREATE INDEX idx_patients_name ON patients(name);
```

#### 5.1.2 patient_visits 表（就诊记录）
```sql
CREATE TABLE patient_visits (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    visit_no VARCHAR(100) UNIQUE NOT NULL, -- HIS就诊号
    patient_id UUID NOT NULL REFERENCES patients(id),
    visit_type VARCHAR(32),      -- outpatient/inpatient/emergency
    department VARCHAR(100),
    doctor_id UUID REFERENCES users(id),
    doctor_name VARCHAR(100),
    visit_date TIMESTAMP WITH TIME ZONE,
    chief_complaint TEXT,
    present_illness TEXT,
    status VARCHAR(32) DEFAULT 'active', -- active/completed/cancelled
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
CREATE INDEX idx_patient_visits_patient ON patient_visits(patient_id);
CREATE INDEX idx_patient_visits_no ON patient_visits(visit_no);
```

### 5.3 检验相关表

#### 5.6.1 audit_alerts 表（审核预警记录）
```sql

    handled_at TIMESTAMP WITH TIME ZONE,
    handle_remark TEXT,

    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
```

### 6.1 规则定义增强

#### 6.1.1 优化 audit_rules 表
**新增字段：**
```sql
rule_type VARCHAR(64)            -- diagnosis_suggest/exam_recommend/medication_warning等

```

### 6.2 规则条件表

#### 6.2.1 rule_conditions 表
```sql
CREATE TABLE rule_conditions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    rule_id UUID NOT NULL REFERENCES audit_rules(id) ON DELETE CASCADE,
    condition_group_no INT DEFAULT 1, -- 条件组号（支持OR逻辑）
    field_name VARCHAR(128) NOT NULL, -- age/gender/diagnosis_code/lab_value等
    operator VARCHAR(32) NOT NULL,    -- >/</=/contains/in/between
    field_value VARCHAR(255),
    value_type VARCHAR(32),           -- string/number/boolean/array
    logic_type VARCHAR(16) DEFAULT 'AND', -- AND/OR
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
CREATE INDEX idx_rule_conditions_rule ON rule_conditions(rule_id);
```

### 6.3 规则动作表

#### 6.3.1 rule_actions 表
```sql
CREATE TABLE rule_actions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    rule_id UUID NOT NULL REFERENCES audit_rules(id) ON DELETE CASCADE,
    action_type VARCHAR(64) NOT NULL, -- alert/recommend_exam/recommend_drug/block_order等
    action_title VARCHAR(255),
    action_content TEXT,
    action_json JSON,                 -- 动作参数
    recommendation_level VARCHAR(32), -- 推荐级别
    allow_apply BOOLEAN DEFAULT true,
    allow_ignore BOOLEAN DEFAULT true,
    ignore_reason_required BOOLEAN DEFAULT false,
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
CREATE INDEX idx_rule_actions_rule ON rule_actions(rule_id);
```

### 6.4 规则适用范围表

#### 6.4.1 rule_scopes 表

```sql
    doctor_scope VARCHAR(255),        -- 适用医生
```
