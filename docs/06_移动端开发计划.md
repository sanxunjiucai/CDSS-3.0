## 一、产品定位

CDSS 移动端是医生**随身携带的临床知识助手**，供医生在查房、会诊、移动场景下快速查询临床知识。功能为 PC 端子集，以**知识查询与工具使用**为核心，不涉及 HIS 系统集成。

**目标用户：** 临床医生、住院医师、规培医生
**核心使用场景：** 查房时查询用药、检验报告参考值、临床量表评分、公式速算

## 二、设计风格定义

### 2.1 参考对象

| 参考产品 | 借鉴要点 |
|--------|---------|
| **蚂蚁阿福** | 对话式搜索入口、结构化卡片、温暖专业的视觉调性、IP亲切感、多功能入口清晰分层 |
| **丁香医生** | 医学内容卡片样式、标签体系、分类导航结构 |
| **医学参考（Epocrates）** | 信息密度适中的详情页、药品速查样式 |
| **默沙东诊疗手册App** | 疾病知识结构化展示、层级清晰 |

### 2.2 设计风格关键词

> **温暖 · 专业 · 高效 · 清晰**

借鉴蚂蚁阿福的核心设计理念：
- **去恐惧化**：医疗信息以友好卡片呈现，减少严肃感，增加信任感
- **对话式入口**：首页以搜索框为视觉中心，天然引导查询行为
- **结构化内容卡片**：信息分层清晰，关键数据突出展示
- **沉浸式工具流**：进入量表/公式等工具时，给予明确的任务场景感

| 角色 | 色值 | 用途 |
|-----|------|-----|
| **主色 Primary** | `#1677FF` | 主要按钮、链接、高亮 |
| **健康绿 Health** | `#00B96B` | 正常值标识、成功状态 |
| **警示橙 Warning** | `#FA8C16` | 偏高/偏低值、注意提示 |
| **危急红 Danger** | `#F5222D` | 危急值、严重警告 |
| **背景 Background** | `#F5F7FA` | 页面底色 |
| **卡片白 Card** | `#FFFFFF` | 卡片底色 |
| **主文字** | `#1C2D4F` | 标题、主要内容 |
| **次文字** | `#6B7FA3` | 副标题、辅助说明 |
| **边框** | `#E8EDF5` | 分割线、卡片边框 |

### 2.4 字体规范

```
标题大  24sp  Bold    页面主标题
标题中  18sp  SemiBold 卡片标题、Section 标题
正文大  16sp  Regular  主要内容文字
正文小  14sp  Regular  描述文字、标签
辅助字  12sp  Regular  时间戳、元数据
```

### 2.5 组件风格

- **圆角**：卡片 `12dp`，按钮 `8dp`，标签 `999dp`（胶囊型）
- **阴影**：`0 2px 12px rgba(22,119,255,0.08)`，轻盈不厚重
- **间距**：基准 `8dp`，主内容区 `16dp` 水平边距
- **图标**：Lucide Icons 线性风格，统一 `24dp` 尺寸
- **过渡动画**：页面切换 `300ms ease-out`，卡片点击 `150ms`

## 四、技术选型

### 4.1 框架选择：React Native (Expo)

| 方案 | 优点 | 缺点 |
|-----|------|-----|
| **React Native + Expo** ✅ | 与现有 React 18 技术栈完全一致，代码复用率高；Expo 开箱即用，无需原生配置；支持 iOS + Android | 包体比原生大 |
| uni-app (Vue3) | 支持小程序 | 与现有 React 技术栈不符，需重新学习 |
| Flutter | 性能最佳 | 全新语言（Dart），投入成本最高 |

**最终选型：React Native + Expo SDK 51**

### 4.2 技术栈

| 层次 | 技术 | 说明 |
|-----|------|-----|
| 框架 | React Native 0.74 + Expo SDK 51 | 跨平台移动端 |
| 样式 | NativeWind v4（Tailwind for RN） | 与PC/管理端 Tailwind 规范统一 |
| 导航 | Expo Router v3（文件路由） | 类 Next.js 体验，URL 路由 |
| 状态管理 | Zustand（与 PC 端统一） | 轻量状态管理 |
| 网络请求 | Axios（与 PC 端共享封装） | 统一 Token 注入、错误处理 |
| 图标 | @expo/vector-icons + lucide-react-native | 统一图标体系 |
| 存储 | AsyncStorage | 本地收藏、浏览历史 |
| 动画 | React Native Reanimated 3 | 丝滑过渡动画 |
| 构建 | EAS Build（Expo Application Services） | 云端构建 iOS/Android |

## 九、开发约定

### 9.1 代码规范
- TypeScript 严格模式，组件使用函数式 + Hooks
- 文件命名：页面用 `PascalCase.tsx`，组件用 `PascalCase.tsx`，工具函数用 `camelCase.ts`
- 样式优先使用 NativeWind Tailwind 类，避免内联 StyleSheet

### 9.2 体验红线
- 列表页必须有骨架屏（Skeleton），不允许纯白闪烁
- 所有网络请求失败必须有友好错误提示，带重试按钮
- 详情页图片/长文字必须处理加载态
- 键盘弹出时不能遮挡关键输入框

### 9.3 性能要求
- 首屏 Tab 切换 < 100ms
- 搜索结果展示 < 500ms（与 PC 端一致）
- 列表页使用 `FlashList`（高性能替代 FlatList）
