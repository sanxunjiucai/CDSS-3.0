## 一、开发目标与范围

### 1.1 本计划目标

基于已完成的项目骨架（后端 FastAPI + 前端目录结构），开发 **医生PC端（独立知识查询端）**，满足磋商文件中"**医疗知识库**"模块的功能要求，具体覆盖：

| 知识库模块 | 磋商文件要求 | 本计划优先级 |
|----------|------------|------------|
| 疾病知识库 | 疾病列表浏览、分类导航、详情查看 | P0 |
| 药品库 | 药品检索、说明书详情 | P0 |
| 检验检查知识库 | 检验项目查询、参考范围、临床意义 | P0 |
| 全局检索 | 跨库统一搜索、关键词/ICD编码检索 | P0 |
| 临床指南库 | 指南列表、全文浏览 | P1 |
| 医学公式库 | 公式列表、参数计算 | P1 |
| 评估量表 | 量表列表与使用入口 | P1 |
| 动态文献库 | 最新文献检索与浏览 | P2 |
| 案例文献库 | 典型病例文献展示 | P2 |
| 三基知识库 | 三基知识查询 | P2 |

### 1.2 技术栈决策

> **统一使用 React 技术栈**，与已完成的 HIS嵌入端保持一致，实现组件复用。

| 技术 | 选型 | 说明 |
|-----|------|------|
| 框架 | React 18 + Vite | 与 HIS端同框架，共享 shared/ 组件 |
| 样式 | Tailwind CSS 3.x | 与 HIS端同 Design Token |
| UI 组件 | Radix UI + 自定义组件 | 与 HIS端同库 |
| 状态管理 | Zustand | 轻量状态，用户偏好/搜索历史 |
| 路由 | React Router v6 | 多页面导航 |
| HTTP | Axios（通过 shared/api） | 复用已有 API 封装 |
| 图标 | lucide-react | 与 HIS端一致 |

## 四、分阶段开发任务

### 第一批（P0 核心）— 工程初始化 + 基础页面

**目标：** PC端可独立运行，实现"搜索 → 结果 → 详情"完整闭环

#### 任务清单

**Step 1：工程初始化**
- [ ] 初始化 `frontend/pc/` 项目（package.json、vite.config.js、tailwind.config.js）
- [ ] 复用 HIS端 Tailwind Design Token（颜色、字体、圆角保持一致）
- [ ] 配置路由（React Router v6）
- [ ] 配置 `@shared` 别名，复用 `frontend/shared/api/`
- [ ] 配置 API 代理（dev server → backend :8000）

**Step 2：全局布局**
- [ ] `PCLayout.jsx`：顶部导航栏 + 左侧折叠导航 + 内容区
- [ ] `TopBar.jsx`：Logo「CDSS」、全局搜索框、用户头像/登出
- [ ] `SideNav.jsx`：知识库分类导航（疾病/药品/检验/指南/公式/量表）

**Step 3：首页工作台**
- [ ] 大搜索框（支持下拉选择知识库类型）
- [ ] 知识库分类导航入口（图标卡片式，9宫格或6宫格）
- [ ] 最近浏览记录（LocalStorage）
- [ ] 更新公告区域（静态占位，后期接口）

**Step 4：全局搜索**
- [ ] 搜索结果页 `/search?q=xxx&type=xxx`
- [ ] 调用 `GET /api/v1/search?q=xxx`
- [ ] 左侧分类过滤（疾病/药品/检验/指南），结果数量展示
- [ ] 每条结果点击跳转对应详情页

**Step 5：疾病知识库**
- [ ] 疾病列表页（支持按科室筛选，分页）
- [ ] 疾病详情页（概述/病因/症状/诊断标准/治疗/预后各节展示）
- [ ] 详情页关联推荐（关联药品、关联检验）

**Step 6：药品库**
- [ ] 药品列表页（支持按分类筛选，分页）
- [ ] 药品详情页（通用名/商品名/适应症/用法用量/禁忌/不良反应/相互作用）

**Step 7：检验检查知识库**
- [ ] 检验/检查列表页（可切换 lab/exam 类型）
- [ ] 检验详情页（参考范围分层表格、临床意义、检查前准备）

**阶段产出：**
- PC端可独立访问（端口 3000）
- 搜索 → 结果 → 疾病/药品/检验详情 完整闭环
- 与后端 API 真实对接（需后端 Seed 数据到位）
- 可演示的知识门户基础形态

## 五、与后端 API 对接关系

PC端消费的 API（均已在后端骨架中定义）：

| 功能 | API | 状态 |
|-----|-----|------|
| 全局搜索 | `GET /api/v1/search?q=&type=` | 骨架已有 |
| 疾病列表 | `GET /api/v1/diseases?department=&page=` | 骨架已有 |
| 疾病详情 | `GET /api/v1/diseases/:id` | 骨架已有 |
| 药品列表 | `GET /api/v1/drugs?category=&page=` | 骨架已有 |
| 药品详情 | `GET /api/v1/drugs/:id` | 骨架已有 |
| 检验检查列表 | `GET /api/v1/exams?type=&page=` | 骨架已有 |
| 检验检查详情 | `GET /api/v1/exams/:id` | 骨架已有 |
| 指南列表 | `GET /api/v1/guidelines?page=` | 骨架已有 |
| 指南详情 | `GET /api/v1/guidelines/:id` | 骨架已有 |
| 量表列表 | `GET /api/v1/assessments?page=` | 骨架已有 |
| 量表评分 | `POST /api/v1/assessments/:id/score` | 骨架已有 |

> **说明：** 后端骨架已定义接口路由，但 Service 层业务逻辑和数据库 Seed 数据需同步推进。

## 六、设计规范（与 HIS端统一）

### 6.1 色彩 Token（沿用 HIS端）

| Token | 色值 | 用途 |
|-------|------|------|
| primary | #1677FF | 主色、链接、按钮 |
| navy | #1C2D4F | 顶部导航背景 |
| bg | #F5F7FA | 页面背景 |
| border | #E4E9F0 | 分割线、边框 |
| danger | #FF4D4F | 错误、禁忌高亮 |
| warning | #FA8C16 | 警告、异常值 |
| success | #52C41A | 正常值 |

### 6.2 PC端布局规格

```
整体宽度：最小 1280px，内容区最大 1200px
左侧导航：240px（可收折为 64px 图标模式）
顶部导航：60px 高度
内容区：padding 24px
卡片：border-radius 8px，shadow-sm
```

### 6.3 字体大小（PC端放大版）

PC端相比 HIS端（紧凑型）适当放大字体：

| 用途 | 大小 |
|-----|------|
| 正文 | 14px |
| 小字 | 12px |
| 标题/Section | 16px |
| 大标题 | 20px |
| 超大标题（首页） | 28px |
